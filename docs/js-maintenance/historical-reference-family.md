# 修改版 `game.js` 的历史官方参考家族

## 1. 结论

公共归档中没有证据能证明某一个官方文件就是修改版 `game.js 1.0.2` 的直接母版。

但通过两层排名，已经把最值得参考的历史范围缩小到：

```text
2025-08-06  官方参数 202508060003
2025-08-20  官方参数 202508200001
2025-09-10  官方参数 202509100001
2025-09-18  官方参数 202509180001
2025-10-17  官方参数 202510170003
```

这五个版本构成当前最可靠的**2025年8—10月官方参考家族**。

## 2. 为什么不能继续按 `1.0.2` 搜索

`1.0.2` 是自托管修改版发布清单中的版本标签，不等于官方 CDN URL 的版本参数。

修改版内部还出现了：

```text
logic version: 1.2.4
```

它同样不能直接等同于官方 CDN 参数。官方归档采用日期型参数，例如：

```text
202509100001
202510170003
```

因此母版判断必须依赖代码结构、接口集合和历史时间，而不是寻找同名 `1.0.2`。

- 文件大小。

## 5. 如何解释低 token 和低块复用

虽然字段、SAL、入口结构和大小高度接近，但：

```text
规范化 token 相似度：约 0.13—0.18
精确 4 KiB 块复用：约 0.3%
```

这说明至少发生了其中一种情况：

1. 修改版重新经过整包混淆；
2. 使用不同的构建器或混淆配置重新生成；
3. 在官方代码基础上进行了较大范围的插桩或改写；
4. 变量、常量、控制流和字符串编码被整体重排；
5. 直接母版可能位于五个归档版本之间，但公共归档没有保存。

因此不能通过普通文本 diff 自动重建修改逻辑。

## 6. 后续维护应如何使用这五个版本

### 首选结构参考

```text
202510170003
```

它在深度指纹中排名第一，适合用于研究同代模块结构、字段和流程。

### 首选体积与语义参考

```text
202509100001
```

它在第一层排名第一，体积最接近修改版，语义字符串相似度在五个候选中最高。

### 参考家族而不是单一母版

后续更新应同时比较五个版本：

- 五版都存在的结构，视为同代稳定结构；
- 只在某一版出现的内容，视为官方短期变化；
- 修改版存在、五版均不存在的内容，才列为“疑似自定义或缺失历史版本差异”；
- 在找到直接来源链前，不把“疑似”写成“确定修改”。

## 7. 2026归档候选的异常

归档索引还包含2026年2月和6月版本，但本轮读取结果没有识别出正常的 SAL/入口结构。

这更可能是：

- 归档内容编码或压缩未被正确还原；
- 归档保存的是包装响应或异常页面；
- Wayback 返回的内容与原响应不同。

不能据此把2026版本归类为旧架构。它们暂时标记为“归档内容不可用于结构比较”。
